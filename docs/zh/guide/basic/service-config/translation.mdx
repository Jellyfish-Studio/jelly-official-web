# 翻译服务

## 列表顺序说明

:::warning 注意
在调整模型服务商列表顺序前，请仔细阅读本部分，避免因意外调整顺序，导致工具调用错模型，给您造成损失。
:::

<MacScreenShot src="/translation-service-list.png" />

翻译服务沿用了[大模型服务列表的设计](/guide/basic/service-config/llm#列表顺序说明)。

与大模型列表不同的是，**_翻译服务列表的第一个服务还会自动用作气泡窗中的翻译_**。

您可以通过[安装插件](/plugin/add-service-plugin)扩展气泡窗翻译服务。但目前插件生态还不成熟，建议您第一个固定先放系统翻译。

## 使用建议

目前对于大多数翻译场景，使用大语言模型（LLM）进行翻译的效果，远比传统的机器翻译要好。如果您仅需要翻译文本，建议优先使用大模型服务。

Jelly 之所以支持翻译类型的服务，主要是为了支持词典类型的输出，像这样：

<img className="img-radius-sm" width={300} src="/dict-res.png" />

这个词典是带音标和发音的。

> 虽然内置工具中，也有使用大模型实现的词典，但是那个没有发音功能。

## 支持服务列表

目前 Jelly 内置仅支持系统翻译服务。您可以通过[安装插件](/plugin/add-service-plugin)来安装其他翻译服务，供气泡窗或“翻译/词典”工具调用。

### 系统翻译

系统翻译使用 Apple Translation API 实现，使用本地的神经网络引擎进行推理，速度极快，效果也还行。最大的问题是不能感知上下文。如果您需要结合上下文翻译，还是要使用“AI 翻译”工具。

首次使用需要在 **设置 -> 语言与翻译** 中下载语言包。
